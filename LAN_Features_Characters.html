Introduction

B4P supports the full UNICODE character set which includes

  • The Basic Multilingual Plane (codes 0 ... 65,535 / U+0000 ... U+FFFF), as well as

B4P treats every UNICODE character as one single character. For example, 'Café' counts 4 characters. The full character set is available to define variable names, table names, table header names, user function names, path and file names, etc. Internally, in order to conserve memory needs for large data, all text data is stored and handled in UTF-8 format, but you don't need to worry about ths.

Note that Latin, Greek and Cyrillic characters share symbols which look identical, e.g. the capital letter 'A'. Attempting to compare them will indicate that they are different. As another example, the Greek mu μ and the micro µ symbols are different, too.

  inhabitants [ Zürich ] = 402000;
  Пётр Чайкoвский [ famous concert ] = Nutcracker; // Piotr Tschaikowski
  echo( inhabitants [ Zürich ] );
  echo( Пётр Чайкoвский [ famous concert ] );

Line Separator Symbols

B4P is able to load text files containing both "carriage return + line feed" (CR+LF) characters common in Windows systems and "line feed only" (LF) common in UNIX/LINUX/MACOS systems. Internally, B4P considers the reserved keyword new line as a single character. When saving contents to files, the new line symbol will be automatically saved as line-feed (LF) in UNIX / Linux / MacOS systems and carriage-return + line-feed (CR+LF) character pairs in Windows systems.

The legacy Macintosh "carriage return only" (CR) serving as line separator is not supported.

Function, Arrow and Other Navigation Keys

B4P has the unique capability to translate characters received with the functions getch() or input quick() into dedicated UNICODE characters situated in the Private Use Area B. This area is freely available for own use and will not interfere with assignments of new characters in the future. Every key pressed, e.g. a function or arrow key, will return a single character.

See Special keyboard input characters.

Note: You may have considered Escape sequences (which date back to Digital Equipment VT100 terminals in the late 1970's , e.g. s ESC [A for a cursor key) but these symbols do typically contain multiple characters (up to seven of them) and B4P will not guarantee cross platform portability.

Special Console Output Characters for Formatting

B4P is also supporting a set of dedicated UNICODE characters for formatting literal contents before outputting the with standard functions such as echo(), format print() and print(). Typical examples are changing text and background color, enabling underscore and some cursor navigation symbols.

See Special console output characters.
