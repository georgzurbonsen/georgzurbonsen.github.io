Introduction

Simple (unquoted) literals can be used for

  • String values
  • Variable names
  • Table names
  • Procedure and function names
  • Reserved keywords (true, false, else, tab, escape, new line)

Rules

Following rules apply for unquoted literals:

  • They may consist of 1 or more words separated by white spaces (space bar, tab, etc.)
  • The literals must fit on the same line
  • The number of consecutive spaces between words are irrelevant. They are always counted as 1 spece.
    • Example: Hello   World is the same as Hello World
  • Leading and trailing spaces are ignored
    • Example: echo(Hello World) outputs the same as echo(  Hello World ).
  • The literals may consist of both text and numbers separated with spaces
    • Example 1: 4200 Pennsylvania Avenue or Number 1
    • Example 2: 555 1212 (phone number is a literal because of the space inside).
    • Example 3: 1.234.567 (multiple decimal points inside makes it a literal
  • All characters are allowed, as long these characters are not relevant for the B4P language syntax.
    • Example: The hyphen in Baden-Baden (a spa city in Germany) is interpreted as a minus sign and does a text subtraction of Baden. The outcome is an empty string. In this case, specify 'Baden-Baden' or "Baden-Baden" instead.
  • A lone point (.) specified without quotation marks is also interpreted as a string containing this one character as long it is not a decimal point in a numeral. ** The lone point is also used to reference a current column in partial table specifications.
    • Note: Two consecutive points is a B4P range symbol and does not consitute a literal.
  • Reserved keywords are automatically treated as such, e.g. true as Boolean true and not as a string.

Examples
  echo( Hello World );
  echo(   Hello   World   );  // Same as above
  echo( One + One );          // + sign is an operator, result is 'OneOne'
  echo( 'One + One' );        // This one is OK.
  echo( Baden-Baden );        // Empty string.  Hyphen is treated as minus sign (subtraction of literals)
  echo( 'Baden-Baden' );      // This one is OK.
  echo( 1. 23 );              // This is a literal because a space is inside.
  echo( 1 200 );              // Same here.  Use function 'clean numeral' to extract 1200 as a number from here
  echo( Dial 555 1212 );
  echo( this is on, new line, the next line );  // new line is a reserved keyword
  echo( Café in Zürich );     // Foreign characters inside
  echo( . );                  // Lone point
  echo( .   . . );            // A string containing 3 lone points (. . .), also showing how spaces are collapsing
  // echo( .. );              // Would cause an error because .. is a b4p language symbol.
  echo( '..' );               // This one is OK
