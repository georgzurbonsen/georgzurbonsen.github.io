Function Names

type, typd detailed

Description

The function type returns one of the following values:

  • numeral
  • literal
  • boolean
  • date
  • parameter set
  • void

The function type detailed also distinguishes between following values:

  • numeral / plain numeral (plain numeral contains no literal representation along with the value)
  • date / plain date (plain date contains no literal representation along with the value)
  • literal / 'literal' / "literal" (stands for literal, softquoted literal, quoted literal)

Call as: function

Restrictions

Indirect parameter passing is disabled

Parameter count

1

Parameters

No.TypeDescription
1
input		 all types Value

Value to be converted as described above.

Return value

Examples

      a[num 1] = 1;
      a[num 2] = a[num 1] + 0;
      a[lit 1] = An unquoted literal;
      a[lit 2] = 'A softquoted literal';
      a[lit 3] = "A quoted literal";
      a[lit 4] = softquoted literal( a[lit 3] );
      a[date] = date(today);
      a[bool] = true;
      a[pset] = { 1,2,3 };
      a[void] = null;

      for all variables( a[], value[], counter[], name[] )
      {
          echo( name[], ": ", value[], " --> ", type( value[] ), ", ", type detailed( value[] ) );
      }

lit 3: A quoted literal --> literal, "literal"
lit 4: A quoted literal --> literal, 'literal'
num 1: 1 --> numeral, numeral
num 2: 1 --> numeral, plain numeral
pset: {1,2,3} --> parameter set, parameter set
