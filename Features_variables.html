Variables

Beyond4P provides variables which can either be simple variables, arrays or structures. Nested variables are allowed, e.g. an array of structures, arrays and simple variables, and a structure of arrays, structures and simple variables. You can build up any tree of variables in order to manage complex data structures efficiently.

Different from other programming languages, and actually for a good reason, all variables are specified with brackets at the end, e.g. variable name[]. Variable names can take any literal value, even in quotation marks. Variable names may consist of multiple words and special symbols, including non-ANSI characters. If the variable name contains special symbols recognized otherwise by Beyond4P (e.g. +, -), or contains multiple consecutive white spaces, then the variable needs to be encapsulated inside single or double quotation marks. Variable names are case sensitive.

Variable Form Description Example
Simple Variables Simple variables one value of any data type (numeral, literal, date, boolean, void and paramter set). last name[]
Arrays Arrays contain a number of array members. The members are referred with index values. member [1], member[3]
Structures Structures contain a number of named member variables animal[dog], animal[cat]
Base Variables Arrays and structure can store, in addition to their member variables, a value in the base variable parent[], parent[son]

Every variable contains following information attributes:

Attribute Description
Variable Name A literal of choice.
Value Stored value (Base Variable)
Type Data type of stored value (numeral, plain numeral, date, plain date, literal, softquoted literal, quoted literal, boolean, void, parmeter set)
Form Distinguishes whetehr the variable is a simple variable, an array or a structure, or a simple variable containing zero members
Members Contains all member variables (arrays, structures)
Protection Setting Applies rules on accessing and/or modifying these variables, e.g. read-only, prevent deleting, etc.
Locked Variables will be locked during the time of executing an assignment to that variable (e.g. prevents deleting the variable while the calculation of the value to be assigend is ongoing).
