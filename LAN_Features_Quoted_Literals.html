Introduction

Quoted literals can be used for

  • String values
  • Variable names
  • Table names

Rules

Following rules apply for softquoted literals:

  • Use double quotation marks " ... " to enclose the text
  • The literals must fit on the same line (unless specified as multiline strings using 3 consecutive quotation marks """)
  • They may consist of 1 or more words separated by white spaces (space bar, tab, etc.)
  • The literal constant may contain multiple consecutive spaces and the number of spaces are relevant
    • Example: "Hello   World" is not equal to "Hello World"
  • Leading and trailing spaces are supported.
    • Example: "Hello World" is not equal to " Hello World "
  • All characters are allowed, except the double quotation mark used to conclude the string constant
  • A lone point ('.') is always treated as a string value, even to specify table headers equalling to "." literally
  • Reserved keywords inside quotation marks are string constants and not reserved keywords, e.g. "true" constitutes a string containing 'true'.
  • Wildcards are not supported. Comparison with A* will actually compare with these contents as given, e.g. ABC = "A*" returns false.
  • Character entities like &euro; or &#x20ac; for the EURO symbol € are not supported here. The contents are taken over as given.

Examples
  echo( "Hello World" );
  echo( "  Hello   World  " );
  echo( "One + One" );
  echo( "'single quotation Marks'" );
  echo( "Price tags: &euro; 10.00, &#x20ac; 10.00, &#8364; 10.00"); // No translation of entities here
  echo( "3 * 2 = 6" );
  echo( "A cheap way to continue on&#10;the next line" ); // No translation of entitites here
