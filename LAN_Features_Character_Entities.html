Introduction

B4P supports a large number of character entities. These are common approaches to define special characters with their names. The key purposes in HTML is to avoid characters like < as HTML tags and other relevant symbols, and also to specify these characters if they are not found on your keyboard.

Most character entities are the same ones as supported in HTML and by web browsers. However, B4P cannot guarantee full coverage of all such entities. The function dump entities() will generate a table with all supported entities.

In addition, B4P provides proprietary character entities to define special console output characters for the console (e.g. text and background color, text attributes like underlining), als well as special keyboard input characters like pressed cursor navigation keys in form of single characters. Click on the links in this paragraph for more details.

Character entities are supported in string constants of type softquoted literal, i.e. the string constant must be specified with single quotation marks (apostrophes) at the beginning and at the text. They will be converted into characters before program execution begins, quasi at compile time. Contents inside double quotation marks will not be converted. The text contents are considered as given.

  echo( 'Value is &ge; 100 &eur;' );  // Softquoted literal used
  echo( "Value is &ge; 100 &eur;" );  // Quoted literal used - will show plain text
  echo( decode entities( "Value is &ge; 100 &eur;" )); // Entities decoded at run-time

  echo( '&#x20ac;'); // EURO Sign - UNICODE hexadecimal code
  echo( '&#8364;' ); // EURO Sign - UNICODE decimal code
