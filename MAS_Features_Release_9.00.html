Overview

Release 9.00, updated 2022-01-10, "Amerigo Vespucci"
Famous salesman, sailor and navigator. And naming origin of "America".

Bugfixes

  • Calling print() alone and without parameters causes a program crash, done by accessing an invalid memory location. This has been fixed.
  • Inside switch() or check() blocks with cases, the else block would be executed even if supposed to skip over it. This has been fixed.
  • Function define additional procedure and function() was refused and returned error message. This has been fixed.

New Functions

  • console(): Read out console parameters such as text and cusor positions as well as console sizes in number of rows and columns
  • underscore(), boldface(), blinking(), reverse(): Various console features. Implementation may vary among different consoles and terminal emulators.
  • format() and format print() generated formatted text output for console use. The latter function outputs the contents directly to the console.
  • print bar() outputs a bar symbol or string across the full width of the visible console
  • distribute amount() distributes a numeric value across multiple values using specific rules, e.g. even, random, etc.
  • weight() measures the weight, i.e. number of members and sub-members of the specified variable
  • weight profile() works similar to weight(), but the weights are recorded separately for every hierarchical level in order to get some tree profile info
  • depth() returns the depth of the specified variable, i.e. maximum nesting level of members and their sub-members
  • table insert rows if needed(), table delete rows if needed(), table delete remaining rows if needed() suppress exceptions in if the specified row number lies below the table.
  • inspect() - New implementation for active visualization of variables
  • see() - Old implementaiton of inspect().
  • dump entities() - Provides a list of all HTML-like entities B4P is supporting.

Improvements in Existing Functions

  • text color() and background color(): In the past, only 16 colors were supported. Various additional color names have been added, and it is possible to provide RGB-colors in hexadecimal format (#rrggbb), as function parameters or three numbers in a parameter set.
  • getch() and input quick() now support arrow, function and other non-character navigation keys.
  • delete members() allows to delete members of a selected range

New Language Features (e.g. syntax and semantics)

  • Various new HTML like entities have been introduced for UNICODE characters in the PUA (Private Use Area) in order to do text formatting and colors on the console, as well as recognizing special keyboard keys such as function and navigation keys.

Other new Features

Interactive mode has been improved where better editing is supported, including accessing previous entered text and accessing the clipboard. As LINUX / MacOS have not advanced input features like in Windows, this improvement is particularly significant for those systems.

Error messages make use of text color for better clarity.

Performance Improvements

table save excel file() runs significantly faster when saving large excel files.

Depreciated and retired features

  • docs search() is now part of docs().
  • inspect() as implemented in the past has changed to see().
