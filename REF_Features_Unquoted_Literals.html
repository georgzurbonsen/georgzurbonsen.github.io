Introduction

Simple (unquoted) literals are used to name variables, functions, tables and reserved keywords.
Attention: Don't forget to add the [] sequence behind the literal if it shall be used as a variable and not as a string value because B4P interpretes stand-alone unquoted literals as string values.

Following rules apply:

  • They may consist of 1 or more words separated by white spaces (space bar, tab, etc.)
  • Multiple consecutive spaces in the literal will collapse to one space symbol (e.g. Hello   World is the same as Hello World)
  • The literals may consist of both text and numbers separated with spaces, e.g. 4200 Pennsylvania Avenue, or Number 1 or 555 1212 (phone number is text because of space inside).
  • The literals must fit on the same line
  • Leading and trailing spaces are ignored
  • All characters are allowed, as long they do not interfere with the B4P programmin syntax. Example: The hyphen in Baden-Baden (a spa city in Germany) is interpreted as a minus sign and does a text subtraction of Baden and the outcome is an empty string. In this case, specify 'Baden-Baden' or "Baden-Baden" instead.
  • Reserved keywords are automatically treated as such, e.g. true as Boolean true and not as a string.

  echo( Hello World );
  echo( Hello   World   );    // Same as above
  echo( One + One );      // + sign is an operator, or specify 'One + One'
  echo( Dial 555 1212 );
  echo( this is on, new line, the next line );  // new line is a reserved keyword
  echo( Café in Zürich );       // Foreign characters inside
