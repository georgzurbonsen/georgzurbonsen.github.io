Installation

A separate presentation is available which guides through the installation of Beyond4P under Windows and LINUX.

Starting Beyond4P from Windows

Start menu Click on the start menu and select "Beyond4P". Beyond4P will look for "main.b4p" or alternatively "main.txt" in the default directory which can be specified.
Desktop icon If activated during installation, then click on the desktop icon. Beyond4P will look for "main.b4p" or alternatively "main.txt" in the default directory which can be specified.
Click on Beyond4P application script file (.b4p) In a file explorer or on the desktop, double-click on a .b4p-file to run it directly. Windows will automatically start Beyond4P with that file when running .b4p files.
Command line Enter C:\program files\beyond4p\Beyond4P.exe. To start a specific Beyond4p application sript, then add the file name as command line parameter. Example: C:\program files\beyond4p\Beyond4P.exe HelloWorld.b4p

Starting Beyond4P from Linux

Command line Enter /usr/local/bin/Beyond4P. To start a specific Beyond4p application sript, then add the file name as command line parameter. Example: /usr/local/bin/Beyond4P HelloWorld.b4p
Shebang If the file contains #!/usr/local/bin/Beyond4P in the 1st row, and the Beyond4P file contains Linux-compliant line breaks (line feeds without carriage return symbols), then Beyond4P will be started automatically as the interpreter when executing the specified file.

Starting Beyond4P from MACOS

The same rules apply as for Linux.

Command Line Arguments

Additional information, for example file names, can be added as further command line arguments. Inside the Beyond4P app, the command line arguments are accessible by referencing the global variable command line arguments[] which is configured as an array of literals.

Command Line Options (Switches)

Following command line options influence Beyond4P startup behavior. These options are not added to the global variable array command line arguments[].

-i Start Beyond4P and go into interactive mode immeidately. You can execute any Beyond4P statement as a command. In addition, you can inspect variables and execute operating system commands. You can start Beyond4P apps by typing start filename.b4p.
-w Suppress Beyond4P welcome text. Also the license info text (e.g. demo mode only) will be hidden. This switch is useful when running Beyond4P in a batch script.
-x Skip the "Press [Enter] to end the program" before Beyond4P is exiting. Useful for embedding Beyond4P in batch scripts.
-I MACOS / LINUX only: Starty Beyond4P in installation mode where the welcoming text is skipped. Not recommended for normal applicaction use.

Invalid command line parameters will provide descriptive error messages and stop Beyond4P.

Command line arguments may be freely ordered and specified as combined or individual parameters. See below.

Examples:
 beyond4p -wx myprogram.b4p  // All 5 variants are equivalent.
 beyond4p myprogram.b4p -wx
 beyond4p myprogram.b4p -xw
 beyond4p -w -x myprogram.b4p
 beyond4p myprogram.b4p -w -x
