Introduction

The B4P language is designed to solve problems with as few clearly understandable statements as possible. The language consists of a small number of building blocks:

Feature Description Example
Statements Can be a procedure call, an assignment, an ad-hoc operation or a transaction.
Multiple statements are separated with semiclons ;
Procedure calls A B4P function is called as a procedure echo( Hello World );
Assignments A value or calculated expression is assigned to a variable or table entry a[] = 3 * 3;
Ad-hoc operations Immediate operation applied on a target variable without doing assignment a[]++;
Transactions Smart assignments among variables including their hierarchical member variables and table rows b[] <== a[];
Expressions Values (constants), variables, table entries, functions, or calculated combination of them using operators and parentheses a[] + 2*abs(b[])
Blocks One or more statements contained in braces { ... } in order to execute them jointly, e.g. after if() functions. { a[] = 1; b[] = 2; }



Like in other programming languages, long statements may take multiple lines of text becase the semicolon is delimiting the statements.

Control Flow Statements

In contrast to other programming languages, the control flow statements such as if(), while() and for() loops and many more are indeed B4P functions which are called as procedures and then decide whether to execute the following statements or block containing statements or not. See also the section on Control Flow Functions.
