Introduction

B4P under Linux is operational immediately after installation. No system reboot is necessary.

As a first step, go to the directory /home/username/b4p (b4p is a sub-directory created in your personal home directory. The path stated above may vary among Linux distributions.) In that directory, you will see fllowing files:

username@domain:~$ cd b4p
username@domain:~/b4p$ ls
'B4P Reference Manual.b4p'   interactive.b4p  'start B4P here.b4p'
'hello world.b4p'            main.b4p
username@domain:~/b4p$

Try Hello World

As a first step, take a look at the B4P program 'hello world.b4p'. Use an editor of choice (I typically prefer the gnu editor called 'gedit') to open the file and take a look at the source code.

(You may wish to use wildcard symbols like * to specify a unique part of the file name only)
username@domain:~/b4p$ gedit hello*
#!/usr/local/bin/b4p

// The statement in the 1st row applies to UNIX/Linux/MacOS only and is called a "Shebang".
// It specifies the language interpreter to use when started as an executable file.
// You can delete this line it if you plan to use B4P under Windows only.

// Shebangs are ignored in Microsoft Windows environments.

echo(Hello World);

Close the editor. The following three commands do the same: Start b4p and run the B4P program "hello world.b4p".

  • b4p "hello world.b4p" to start B4P explicitly with the B4P program provided as command line parameter,
  • ./"hello world.b4p" to start the B4P file like an executable program,
  • ./hello*, using wildcard symboles allowing you to specify a part of the file name which is unique in the current directory.

username@domain:~/b4p$ ./hello*

Starting B4P Without a B4P Program Name

You can also start B4P without specifying a file name. In this case, it will look for the B4P program called "main.b4p" in the currently configured starting directory. After the installation with the personal folder created, the directory you are inside is presently the memorized starting directory.

You can change the starting directory to a different location by making a copy of start B4P here.b4p into a different directory and execute this one. Don't do it for this first example yet.

In the starting directory, B4P will look for the file 'main.b4p'. If the file is available, then you will be asked whether to start that one or enter interactive mode instead. Start the program. In case no 'main.b4p' is available, then B4P enters into interactive mode directly.

username@domain:~/b4p$ b4p
(User interaction: Press 'S' to start the program, and type 'exit' to leave interactive mode)

Accessing the Reference Manual

Congratulations. You have been able to get your first B4P programs running. You also find the file "B4P Reference Manual.b4p". Type one of the following commands to open the on-line manual

  • "./B4P Reference Manual.b4p" (Command fully spelled out)
  • ./*Manual* (More convenient if you use wildcard symbols)
  • Inside B4P running in interactive mode, you can type docs to access the documentation, too.

At the first time, you will be asked for the browser you wish (currently Chrome and Firefox are supported), following this the on-line documentation opens up.
