Introduction

The B4P language is designed to solve problems with as few clearly understandable statements as possible, allowing to keep the number of statements, variables and loops as low as possible.

Basic Elements

The B4P language builds on the following basic elements:

  • Text
  • Numbers
  • Language specific symbols (e.g. operators, assignment symbols)
  • Comments

Top-Down Syntax

The top-down syntax and semantic structure is defined as follows:

Language Features Description Examples
Program The program refers to the complete code in one file loaded and executed. It may contain any number of blocks and statements.
Blocks Blocks refer to code encapsulated with braces { ... } and contain any number of statements and nested blocks. Blocks are useful to include all statements in loops, after if() clauses, etc. { a[] = 1; b[] = 2; }
Statements Can be a procedure call, an assignment, an ad-hoc operation or a transaction. Every statement ends with a semicolon. ;
Procedure Calls B4P functions called as procedures. In B4P, control flow statements like if() and while() also procedure calls. echo( Hello World );
Assignments A value or calculated expression is assigned to a variable or to a location in a table. a[] = 3 * 3;
Ad-hoc Operations Immediate operation applied on a target variable without doing assignment a[]++;
Transactions Smart assignments among variables including their hierarchical member variables and table rows b[] <== a[];
Expressions Calculation or combination of the following which return a value: numbers, text, variables, table references, further function calls, and calculation using unary operators and binary operators and parentheses. a[] + 2*b[] + sin( c[] )
Function Calls A B4P function is called and returns a value. It can be used as an expression or part of it. sin( c[] )

Like in other programming languages, statements may be written in multiple lines where no special symbol is needed to indicate that the statement is not yet complete and continues on the next line. The final semicolon ; concludes the statement. The following statement or block may continue on the same line or further below.

Control Flow Statements

In contrast to other programming languages, the control flow statements such as if(), while() and for() loops and many more are indeed B4P functions which are called as procedures and then decide whether to execute the following statements or block containing statements or not. See also the section on Control Flow Functions.
