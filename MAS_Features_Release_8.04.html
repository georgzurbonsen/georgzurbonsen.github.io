Overview

Release 8.04, updated 2021-07-28, "The Big Myth"
The Big Fact: Mountain in Central Switzerland (Grosser Mythen), elevation: 1,898 meters, 47 deg 01' 47" North, 8 deg 41' 19.7" East (695034/209499).

Bugfixes

  • Fixed an old syntax parsing bug addressing sequences where syntax errors have been raised on correct syntax. It affects following code sequence inside switch() and check() sequences.
    case (...) if (...) { ... }
    case (...) ...
    In detail, the following valid case statement has been rejected.
  • Fixed another old syntax parsing bug addressing sequence where syntax errors ahve been raised on correct syntax. It affects loops (all kind of loops) inside if().
    if (...) while (...) { ... }
    else ...
    In detail, the following else keyword has been rejected.
  • If exception() is called inside a user-defined procedure or function in oder to assert a user-defined exception and requesting a calling hierarchy of 1 or higher in order to show the code context of the callign code, then you may need to type "exit" repeatedly to leave B4P when done inspecting. Normally typing "exit" only once shall suffice. This has been fixed.
  • The function call() to call any B4P function by name required 2 parameters all time though the documentation also allows just 1 parameter. This has been fixed.
  • Function table save excel file(): Improper saving of certain formats and style (incl. column width, row height) if 2 or more sheets are saved in one file.

New Functions

None in this release.

Improvements in Existing Functions

  • table load() and table save(): If the file ends with '.xlsx', '.xlsm' or '.xlsb', then the user will be notified that this is the wrong function to load or save Excel files. The error messsage advises to use table load excel file() or table save excel file() instead.
  • table load excel file(): You can load multiple sheets from one Excel file into multiple tables using one single function call.

New Language Features (e.g. syntax and semantics)

  • No longer to explicitly call include(Style Library) in order to add style features to Excel files. This library will be loaded in addition to the other standard libraries: Utility Library, Office Library, Zip Library and Support Library.
  • Calling functions like translate style attributes for html() and translate style attributes for excel xml 2003() explicitly before saving the table into a HTML or old Excel format is no longer needed. These functions will be called automatically before saving, which is already the case with the new Excel format using table save excel file().

Performance Improvements

See improvement on table load excel file(): Loading multiple sheets with one call is faster than calling this function repeatedly.

Depreciated and retired features

None in this release.
