Overview

Release 8.02, issued 2021-04-18, "Édouard Albert Roche"
French Mathematican, deceased 1883-04-18, developed the mathematical theoretic model how the Saturn rings were created.

Bugfixes

  • [table name: column, ''] did not match rows with blank entries if the row width is shorter than column position. Has been fixed.
  • [table name: { column names, ... }] referring multiple column names in parameter sets was rejected when calculated for 2nd time. Has been fixed.
  • Comparing parameter sets with equal to (=) and not equal to (<>) operators: Contents were accidentally compared with the same operator, resulting in unexpected results like {a,'b,c'} = {a,'b,c'} returning false, but {a,'b,c'} == {a,'b,c'} returning true because the softquoted literal 'b,c' is interpreted as equals to b or c. This has been fixed, i.e. contents are compared with strictly equal to approach. The same applies for the <> operator on parameter sets.
  • table copy table columns() and similar functions: Erroneous copying happening when rows in source table are shorter than column specified leads to crashes when accessing these columns afterwards. Has been fixed.
  • Empty blocks, e.g. if (a[]=1) {} have asserted syntax errors which are not. Effective in this new release, empty blocks are allowed.

New Functions

  • fy() and fq() to calculate fiscal years and quarters.
  • table manipulate selected rows() added, works similar like table manipulate(), but on selected rows only.
  • describe() describes the contents, e.g. character, word, line, paragraph, date, time, date time, integer, numeral, scientifc, white and blank.
  • table describe() describe the contents of a table, using a combination of describe() and looking up descriptions in a different table.
  • copy() applies a value in multiple destinations which can be variabls and/or table entries.

Improvements in Existing Functions

  • table manipulate() and table manipulate selected rows(): 1 optional parameter added to decide whether to stop processing the remaining entries in the lookup table or not in case the corresponding statement has been executed.
  • table load excel file() now supports loading Excel formulas into a 2nd table

New Language Features (e.g. syntax and semantics)

  • Multiline strings are supported where a string constant may contain multiple lines of text.

Performance Improvements

  • Minor performance improvements when processing parameter sets (skipped and eliminated internal execution steps no longer needed).
  • Batch-mode installation implemented.

Depreciated and retired features

  • See Release 8.01
