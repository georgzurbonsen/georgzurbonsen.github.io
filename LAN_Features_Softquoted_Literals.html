Introduction

Softquoted literals can be used for

  • String values
  • Variable names
  • Table names

Rules

Following rules apply for softquoted literals:

  • Use single quotation marks ' ... ' to enclose the text
  • The literals must fit on the same line (unless specified as multiline strings using 3 consecutive quotation marks ''')
  • They may consist of 1 or more words separated by white spaces (space bar, tab, etc.)
  • The literal constant may contain multiple consecutive spaces and the number of spaces are relevant
    • Example: 'Hello   World' is not equal to 'Hello World'
  • Leading and trailing spaces are supported.
    • Example: 'Hello World' is not equal to ' Hello World '
  • All characters are allowed, except the single quotation mark used to conclude the string constant
  • A lone point ('.') is also used to reference a current column in partial table specifications.
  • Reserved keywords inside quotation marks are string constants and not reserved keywords, e.g. 'true' constitutes a string containing 'true'.
  • Various functions and comparisons support wildcard symbols like 'A*' for beginning with A, e.g. ABC = 'A*' returns true.
  • Character entities like &euro; or &#x20ac; for the EURO symbol € are supported. They are converted quasi at compile time to target characters.

Examples
  echo( 'Hello World' );
  echo( '  Hello   World  ' );
  echo( 'One + One' );
  echo( '"In double quotation Marks"' );
  echo( 'Price tags: &euro; 10.00, &#x20ac; 10.00, &#8364; 10.00');
  echo( '3 * 2 = 6' );
  echo( 'A cheap way to continue on&#10;the next line' ); // Not recommended. Use '...', new line, '...' instead.
  echo( new line, 'Line 1', new line, 'Line 2' );
